---

This repository is a standalone Next.js/Fumadocs documentation site. Railway can deploy it directly from GitHub.

## Railway service

1. Open Railway.
2. Create a new project.
3. Choose **Deploy from GitHub repo**.
4. Select `Rarex224/docs-hackless-dev`.
5. Let Railway detect the Next.js app.

The default commands are enough:

```bash
npm install
npm run build
npm run start
```

If Railway asks for explicit settings:

| Setting | Value |
| --- | --- |
| Build command | `npm run build` |
| Start command | `npm run start` |
| Node version | `20` or newer |

## Custom domain

Add this custom domain to the Railway service:

```txt
docs.hackless.dev
```

Then create the DNS record Railway shows you. It will usually be a `CNAME` record from:

```txt
docs
```

to Railway’s generated target.

After DNS propagates, Railway provisions HTTPS automatically.

## Recommended production checks

After deployment, verify:

```txt
https://docs.hackless.dev
https://docs.hackless.dev/docs
https://docs.hackless.dev/openapi/openapi.json
https://docs.hackless.dev/openapi/openapi.yaml
```

## Notes

The docs site does not need the Hackless app database. It is static documentation plus a Next.js search route generated by Fumadocs.
